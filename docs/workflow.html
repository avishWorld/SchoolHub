Feedback Loops · מנגנוני היזון חוזר

↺ Feedback Loops — מה חוזר לאן

QA → DEV
באג נמצא → תיקון
QA מוצא באג, כותב Bug Report מובנה, DEV מתקן. חוזר ל-QA לאימות. ללא Sign-off אין DONE.
CTO Review → DEV
Code Review → שיפור
CTO מחזיר קוד עם feedback מפורט. DEV מתקן, מגיש מחדש. מחזור עד שהקוד עובר Review.
DEV → CTO
Blocker → הכרעה
DEV מוצא אי-בהירות בדרישות. מדגל מיד, לא מנחש. CTO או FOUNDER מבהירים, DEV ממשיך.
CTO → FOUNDER
FLAG החלטה בלתי-הפיכה
CTO מציג options + tradeoffs. FOUNDER בוחר. מתועד ב-DECISIONS.md עם date + rationale.
Sprint Review → Sprint Planning
Retrospective Loop
כל ספרינט מסתיים ב-review. מה עבד? מה לא? Velocity? Risks? הלמידה משפיעה על הספרינט הבא.
Production → PRD
User Feedback → PRD Update
פידבק מהורים/מורים/תלמידים משפיע על PRD. PRD.md הוא living document — מתעדכן כשמשהו משתנה.
